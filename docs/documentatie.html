CaT - Camping Info Web Tool

CaT este o aplicatie Web modulara pentru rezervarea, administrarea si compararea locurilor de camping. Aplicatia este dezvoltata in PHP, fara framework-uri, foloseste SQLite pentru persistenta datelor si expune functionalitatile principale prin servicii Web consumate asincron cu Ajax/fetch.

1. Introducere

Scopul proiectului CaT este realizarea unui instrument Web care permite rezervarea si gestionarea locurilor de camping. Aplicatia pune la dispozitie un catalog public de campinguri, pagini individuale pentru fiecare locatie, rezervari, recenzii, mesaje in comunitate si o harta interactiva bazata pe OpenStreetMap.

Solutia este orientata spre partea de server, folosind PHP si SQLite. Clientul este realizat cu HTML, CSS si JavaScript vanilla. Nu sunt folosite framework-uri Web pe partea de server sau client. Aplicatia este structurata intr-o arhitectura MVC simpla si comunica prin endpoint-uri API asincrone.

PHP fara framework SQLite HTML/CSS/JS vanilla Ajax prin fetch() OpenStreetMap GitHub OAuth

4. Arhitectura aplicatiei

CaT este organizata in arhitectura MVC. Directorul app/Models contine modelele pentru interactiunea cu baza de date, app/Controllers contine controllerele care proceseaza cererile API, iar app/Views contine paginile si template-urile de prezentare. Punctul principal de intrare este index.php, iar endpoint-urile Web se afla in directorul api/.

7. Autentificare, OAuth si autorizare

7.1 Autentificare locala

Aplicatia permite crearea unui cont local cu nume, email si parola. Parolele sunt salvate folosind hashing, iar login-ul verifica parola prin mecanismele standard PHP pentru parole.

7.2 GitHub OAuth

Autentificarea OAuth foloseste GitHub. Fluxul incepe prin generarea unui state aleator, salvat in cookie HttpOnly. Dupa autentificarea pe GitHub, callback-ul verifica state, obtine access token-ul si datele utilizatorului, apoi creeaza sau actualizeaza utilizatorul local.

7.3 Bearer token intern

Dupa autentificare, aplicatia genereaza un token intern. Clientul il salveaza in localStorage si il trimite catre API-uri in headerul Authorization: Bearer.

7.4 RBAC - roluri si permisiuni

Sistemul de autorizare este bazat pe roluri si permisiuni. Rolul admin are acces complet. Rolurile custom pot primi permisiuni precum manage_campings, manage_reservations, manage_users, view_stats, import_export si manage_roles.

8. Interfata si design responsive

Interfata este construita cu HTML si CSS, fara framework-uri. Designul foloseste o tema vizuala inspirata de natura, cu nuante de verde, alb si bej. Layout-ul foloseste CSS Grid si Flexbox, iar pentru ecrane mici continutul se reorganizeaza pe o singura coloana.

11. Concluzii si dezvoltari viitoare

CaT acopera cerinta principala printr-o platforma Web functionala pentru campinguri, cu rezervari, recenzii, comunitate multimedia, harta interactiva, administrare, import/export si statistici. Aplicatia respecta constrangerile de implementare: fara framework-uri, cu servicii Web, Ajax, SQLite, HTML/CSS validabile si masuri de securitate de baza.

Dezvoltari viitoare

  • Sistem de notificari pentru rezervari confirmate sau anulate.
  • Moderare completa pentru mesaje si recenzii, folosind permisiunea moderate_messages.
  • Calendar vizual pentru disponibilitatea campingurilor.
  • Export suplimentar in XML sau iCalendar.
  • Optimizare pentru volume mai mari de date si paginare server-side.
  • Testare automata pentru endpoint-uri critice.
